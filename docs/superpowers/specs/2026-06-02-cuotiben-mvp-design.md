# 错题本 Flutter App MVP 产品设计草稿

状态：MVP 设计修订中，等待最终审阅确认
日期：2026-06-02

## 1. 产品定位

这是一款面向初高中理科生及其家长代操作场景的手机错题本管理软件，使用 Flutter 开发 Android 和 iOS 双端应用。

产品核心不是普通 OCR 错题本，而是解决纸质试卷错题整理时无法可靠保留示例图、几何图、物理示意图、公式和原题版式的问题。MVP 的目标是让用户拍照收集错题后，能得到接近原题的可打印练习稿，去掉学生已经写过的手写答案、批改痕迹和草稿内容，方便重新打印做题。

MVP 的核心用户画像是初高中数学、物理错题整理用户。小学生、大学生和非理科科目不作为 MVP 的设计目标，不为这些人群和科目做 UI、OCR、AI 解析或打印版式上的设计妥协。后续版本可以扩展到更广泛学段和科目。

## 2. 已确认的 MVP 决策

- 首版采用“图片主数据 + OCR 辅助”的架构。
- 题目以清理后的题图作为打印和复习的主数据，避免几何图、物理图、公式和版式在 OCR 转写中丢失。
- OCR 使用类似 PaddleOCR 的能力，主要用于题号、文字摘要、搜索、分类、AI 解析 prompt 和辅助排版。
- OCR、自动题框、自动去手写、AI 解析由云端提供。
- 云端处理不能阻塞用户继续整理错题。上传慢、OCR 失败或自动清理失败时，用户仍然可以手动框题、入库、橡皮擦修复和打印。
- MVP 不做账号和云端错题库，错题数据保存到手机本地。后续版本再考虑账号、云同步和多设备能力。
- MVP 聚焦初高中数学和物理，不做中小学全科优化。数学几何图、物理示意图、公式题作为核心验收样例。
- MVP 支持 Android 和 iOS。
- 打印默认输出最终修复题图，并按题型决定是否增加作答区：选择题和填空题不额外留空，解答题下方留作答区；不默认打印答案解析。
- 错题详情提供“答案解析”按钮，由 AI 自动生成解析。
- MVP 不实现图片编辑器、笔刷、线图覆盖或复杂几何图/物理图矢量重绘，只保留橡皮擦功能，用于清除题图中残留的作答痕迹。

## 3. 主流程

1. 用户拍照试卷原图，或从相册导入试卷图片。
2. 用户选择单题模式或多题模式。
3. 单题模式下，用户调整一个题目框；多题模式下，系统尽量自动识别多道题框，用户勾选错题并可调整题框。
4. 用户选择错题后，错题先进入本地题库，保存原始题图。
5. 系统在题库内异步处理错题：图片显示效果增强、PaddleOCR 识别、自动去除手写笔记和答案。
6. 用户进入错题详情页，对每个题目进行人工修复。MVP 只提供橡皮擦，用于清除自动去手写后仍残留的作答痕迹。
7. 用户在错题详情页点击答案解析按钮，查看 AI 生成的答案和解题步骤。
8. 用户从错题库选择多道题，进入打印组卷页，按题型设置解答题作答区并导出 PDF。

## 4. 页面结构

### 4.1 首页 / 拍照入口

- 进入相机拍试卷。
- 选择单题模式或多题模式。
- 支持从相册导入试卷图片。

### 4.2 错题选择页

- 单题模式：用户调整一个题目框。
- 多题模式：系统尽量自动框出多道题，用户勾选错题并可拖拽调整题框。
- 即使云端识别慢，也允许用户手动框选并入库。

### 4.3 错题库

- 按学科、年级、时间、标签筛选。MVP 学科只针对数学、物理优化。
- 每道错题展示缩略图、题号/OCR 摘要、处理状态。
- 处理状态包括：待增强、处理中、已清理、需人工修正、解析已生成。

### 4.4 错题详情 / 橡皮擦修复页

- 展示原始题图、系统增强清理图、用户最终修复图。
- MVP 不实现完整图片编辑功能。
- 只保留橡皮擦功能，用于清除题图中残留的作答痕迹、批改痕迹或草稿痕迹。
- 提供答案解析按钮，调用 AI 生成或查看解析。

### 4.5 打印组卷页

- 用户从错题库选择多道题。
- 按题型决定是否增加作答区：选择题和填空题不额外留空，解答题下方留作答区。
- 支持设置解答题作答区大小。
- 导出 PDF：最终修复题图 + 解答题作答区。
- 默认不打印答案解析。

### 4.6 设置页

- 配置初中/高中、年级、默认学科、打印纸张尺寸。
- 后续版本再加账号、云端同步、会员或家长协作。

## 5. 图像版本原则

题库中每道题至少保留三份图：

- 原始题图：从试卷原图裁剪出来的题目原始图片，用作回退和重新处理依据。
- 系统增强/清理图：云端或本地处理后的题图，包括黑白增强、去阴影、对比度增强、自动去手写。
- 用户最终修复图：用户通过橡皮擦清除残留作答痕迹后的最终打印版本。

打印默认使用用户最终修复图。如果用户没有做橡皮擦修复，则使用系统增强/清理图；如果系统处理失败，则允许使用原始题图经过本地增强后的版本。

## 6. MVP 图像修复边界

MVP 不承诺复杂数学几何图和物理图的完整矢量复原。原因是拍照畸变、细线、虚线、角标、箭头、受力方向、手写覆盖和题干标注会混在一起，自动矢量复原容易生成“干净但错误”的图。

MVP 采用以下策略：

- 保留原图中的题目图形作为主内容。
- 系统通过云端或本地处理完成增强、去阴影、黑白化和自动去手写。
- 用户只使用橡皮擦清除仍然残留的手写答案、批改痕迹或草稿痕迹。
- MVP 不提供恢复笔刷、自由画笔、文字标注、直线、箭头、圆、矩形、裁剪、旋转等图片编辑工具。
- 如果橡皮擦误删题目内容，MVP 允许用户放弃本次修复并回到系统增强/清理图重新擦除。

## 7. 本地数据模型

MVP 本地数据采用“图片文件 + SQLite 元数据”的方式。图片文件保存在 App 本地文件目录，SQLite 只保存必要元数据。

MVP 只建 4 张核心表：

- MistakeQuestion：错题主表，包含图片路径和状态。
- OcrResult：OCR 结果。
- AiSolution：AI 答案解析。
- PrintSet：打印记录。

试卷原图只作为本地图片文件保存，不单独建表；MVP 不提供按原试卷回溯错题的功能。橡皮擦修复不持久化图层或操作历史，用户修复后直接生成最终修复图路径并写回错题主表。

### 7.1 MistakeQuestion

表示一道已进入错题库的错题，也是本地数据的主表。

建议字段：

- id：本地唯一 ID。
- sourceType：camera 或 gallery。
- sourcePaperImagePath：试卷原图本地路径。只存路径，不建试卷表。
- originalQuestionImagePath：从试卷原图裁剪出的原始题图路径。
- enhancedImagePath：系统增强图路径，可为空。
- cleanedImagePath：自动去手写清理图路径，可为空。
- finalRepairedImagePath：用户橡皮擦修复后的最终修复图路径，可为空。
- subject：学科。MVP 只针对 math、physics 做优化。
- gradeLevel：初中或高中。
- grade：年级。
- questionNumber：题号，可由 OCR 辅助识别，也允许用户手动修改。
- questionType：题型，支持 choice、fill_blank、short_answer、essay、unknown。选择题和填空题打印时不额外留空，short_answer 和 essay 视为解答题，需要作答区。OCR/AI 可以给初始值，用户手动修改时直接覆盖该字段，不单独维护来源状态。
- cropRect：题目在试卷原图中的裁剪区域。
- tags：标签列表。
- note：用户备注。
- processingStatus：待增强、处理中、已清理、需人工修正、解析已生成等状态。
- createdAt / updatedAt：创建和更新时间。

### 7.2 OcrResult

表示 PaddleOCR 类能力返回的 OCR 结果。

建议字段：

- id：本地唯一 ID。
- questionId：关联错题 ID。
- rawText：OCR 合并文本。
- blocksJson：文本框、置信度、坐标等结构化结果 JSON。
- confidence：整体置信度。
- detectedQuestionNumber：识别出的题号。
- detectedQuestionType：识别出的题型。识别后可写入 MistakeQuestion.questionType，用户手动修改时直接覆盖。
- engine：OCR 引擎名称和版本。
- createdAt：生成时间。

### 7.3 AiSolution

表示 AI 生成的答案解析。

建议字段：

- id：本地唯一 ID。
- questionId：关联错题 ID。
- promptVersion：使用的 prompt 版本。
- modelProvider：例如 doubao。
- modelName：模型名称。
- answerText：最终答案。
- explanationMarkdown：解析内容，支持 Markdown。
- status：generating、ready、failed。
- createdAt / updatedAt：创建和更新时间。

### 7.4 PrintSet

表示一次组卷打印记录。

建议字段：

- id：本地唯一 ID。
- title：打印标题。
- questionIds：题目 ID 列表，保持用户选择顺序。
- paperSize：A4 等纸张设置。
- answerAreaMode：解答题作答区大小设置，支持 small、medium、large、custom。
- answerAreaRules：作答区规则 JSON，记录哪些题型需要留作答区以及自定义高度。
- pdfPath：生成 PDF 的本地路径。
- createdAt：创建时间。

## 8. 云端接口设计

云端处理采用异步任务模式。App 上传图片后拿到 jobId，可以继续本地操作；任务完成后 App 下载处理结果并保存到本地。

### 8.1 试卷题区检测

`POST /process/paper-detect`

用途：上传试卷原图，返回多题识别的题框建议。

请求：

- image：试卷图片文件。
- mode：single 或 multiple。

返回：

- jobId：异步任务 ID。
- status：queued。

任务结果：

- boxes：题目框列表，每个框包含 x、y、width、height、confidence、suggestedQuestionNumber。

### 8.2 单题图片增强

`POST /process/question-enhance`

用途：上传单题图，返回黑白增强、去阴影、对比度增强、清晰化后的图片。

请求：

- image：单题图片文件。
- options：增强选项，例如 grayscale、contrast、shadowRemoval、deskew。

返回：

- jobId：异步任务 ID。
- status：queued。

任务结果：

- enhancedImageUrl：增强图下载地址。
- metrics：清晰度、对比度等辅助指标。

### 8.3 自动去除手写和批改痕迹

`POST /process/question-clean`

用途：上传单题图，自动去除学生手写答案、批改痕迹和草稿内容，生成可打印的清理版题图。

请求：

- image：单题图片文件。
- preservePrintedText：默认 true，要求尽量保留印刷题干、公式和示例图。
- removeTypes：handwriting、redMark、scratch 等。

返回：

- jobId：异步任务 ID。
- status：queued。

任务结果：

- cleanedImageUrl：清理图下载地址。
- maskImageUrl：识别出的清理遮罩下载地址，可用于错题详情页展示和橡皮擦修复。
- confidence：清理结果置信度。

### 8.4 OCR 识别

`POST /ocr/question`

用途：使用 PaddleOCR 类能力识别题目文字、题号和文本框。

请求：

- image：单题图片文件，优先使用增强图。
- subject：可选学科，MVP 只针对 math、physics 优化。
- grade：可选年级，MVP 只针对初高中年级优化。

返回：

- jobId：异步任务 ID。
- status：queued。

任务结果：

- rawText：合并文本。
- blocks：文本框、坐标、置信度列表。
- detectedQuestionNumber：题号。
- confidence：整体置信度。

### 8.5 AI 答案解析

`POST /ai/solution`

用途：根据题图和 OCR 文本生成答案解析。

请求：

- questionImage：题目图片，优先使用增强图或最终修复图。
- ocrText：OCR 文本。
- subject：学科，MVP 只针对 math、physics 优化。
- grade：年级或学段，MVP 只针对初高中年级优化。
- outputStyle：面向初高中学生和家长的解析风格。

返回：

- jobId：异步任务 ID。
- status：queued。

任务结果：

- answerText：答案。
- explanationMarkdown：分步解析。
- warnings：模型不确定或题图信息不足时的提示。

### 8.6 查询异步任务

`GET /jobs/{jobId}`

用途：查询云端任务状态和结果。

返回：

- jobId：任务 ID。
- type：任务类型。
- status：queued、running、succeeded、failed。
- progress：进度。
- result：任务完成后的结果对象。
- error：失败原因。

## 9. 云端处理边界

- 云端图片只用于临时处理，处理结果下载到 App 本地后，后续版本需要提供自动删除策略和隐私设置。
- 云端 OCR、自动清理和 AI 解析失败时，不阻塞用户橡皮擦修复、入库和打印。
- OCR 结果只作为辅助信息，不作为打印主数据。
- 自动去手写结果必须允许用户通过橡皮擦清除残留作答痕迹。

## 10. PDF 打印版式

MVP 默认采用“练习卷模式”，目标是打印出来像一份可重新练习的错题卷，而不是错题资料页。

### 10.1 默认版式

- 默认纸张为 A4 纵向。
- 每道题使用用户最终修复图。
- 如果用户没有最终修复图，则使用系统增强/清理图。
- 如果系统处理失败，则使用原始题图经过本地增强后的版本。
- 题图按比例缩放，宽度适配页面内容区，不拉伸变形。
- 按题型决定是否增加作答区，符合普通试卷的排版逻辑。
- 选择题和填空题不在题目下方额外留空；题图中原本的选项、括号、填空横线会随题图保留。
- 解答题在题图下方增加作答区。
- 解答题作答区支持小、中、大、自定义四种设置。
- 页眉显示打印标题、学科、生成日期。
- 页脚显示页码。

### 10.2 打印内容规则

- 默认不打印答案解析，避免学生重新练习时直接看到答案。
- 答案解析只在错题详情页通过“答案解析”按钮查看。
- 后续版本可以增加“附答案解析页”的可选导出模式。
- MVP 不把 OCR 文本重排成纯文字试卷。
- MVP 不强行压缩所有题到同一页，优先保证题图清晰和作答空间。
- 题型可以由 OCR/AI 自动识别，但用户必须可以在错题详情或打印组卷前手动修改题型。

### 10.3 组卷交互

- 用户从错题库选择多道题进入打印组卷页。
- 用户可以调整题目顺序。
- 用户可以设置本次 PDF 的标题、纸张尺寸、解答题作答区大小。
- 生成 PDF 后保存到本地，并调用系统分享/打印能力。

## 11. Flutter 技术架构

MVP 使用 Flutter 开发 Android 和 iOS 双端应用。客户端优先保证拍照、入库、橡皮擦修复、打印链路稳定，云端能力作为异步增强。

### 11.1 目录结构

采用 feature-first 目录结构，按业务模块组织代码：

- capture：拍照、相册导入、单题/多题模式选择、题框选择。
- library：错题库列表、筛选、标签、处理状态。
- repair：错题详情、橡皮擦修复。
- print：组卷、打印设置、PDF 生成和系统分享/打印。
- settings：初中/高中、年级、默认学科、打印纸张等设置。
- core：路由、主题、网络、数据库、文件存储、通用工具。

### 11.2 状态管理和路由

- 状态管理使用 Riverpod。
- Riverpod 负责页面状态、异步处理任务状态、本地数据库查询结果和 API 调用状态。
- 路由使用 go_router。
- 主要路由包括首页、拍照页、错题选择页、错题库、错题详情/橡皮擦修复页、打印组卷页、设置页。

### 11.3 本地存储

- 本地数据库使用 Drift + SQLite。
- 数据库保存错题元数据、OCR 结果、AI 解析和打印记录。橡皮擦修复完成后只保存最终修复图路径，不持久化图层或操作历史。
- 图片文件不直接写入数据库，使用 path_provider 管理 App 本地目录，数据库只保存文件路径。
- 每道题至少保存原始题图、系统增强/清理图、用户最终修复图三类图片版本。

### 11.4 拍照和导入

- 拍照使用 camera。
- 相册导入使用 image_picker。
- 拍照或导入后先保存试卷原图，再进入单题/多题模式选择和错题框选。

### 11.5 图片处理和橡皮擦修复

- 题目框选阶段支持调整题框，用于从试卷原图中截取错题区域。
- 系统增强、去阴影、黑白化、自动去手写和 OCR 由云端完成。
- MVP 不实现完整图片编辑器。
- 客户端只实现橡皮擦修复，用于清除自动处理后仍残留的作答痕迹。
- 橡皮擦修复生成新的最终修复图，不直接破坏原始题图或系统增强/清理图。

### 11.6 PDF 和打印

- PDF 生成使用 pdf。
- 系统分享和打印使用 printing。
- PDF 版式遵循“练习卷模式”：最终修复题图 + 按题型生成的解答题作答区。

### 11.7 网络和云端任务

- 网络请求使用 dio。
- 客户端封装 API client，用于上传图片、创建异步任务、轮询 job 状态、下载处理结果。
- MVP 可以先实现 mock service，保证 Flutter 端完整流程可开发和测试；后端后续接入 PaddleOCR、图像处理服务和豆包 AI。

## 12. AI 答案解析格式

错题详情页提供“答案解析”按钮。MVP 中答案解析由 AI 自动生成，后端可以接入类似豆包 AI 的大模型，并通过 prompt 控制输出为初高中学生和家长能接受的解题步骤。

### 12.1 输出结构

AI 解析结果使用 Markdown，建议包含以下结构：

- 答案：直接给出最终答案。
- 解题思路：用学生能理解的话说明解题方向。
- 详细步骤：分步骤推导，数学、物理等科目保留公式。
- 易错点：指出这道题容易错在哪里。
- 同类题方法：可选，给出简短方法归纳。
- 不确定提示：当题图不清、题干缺失或 OCR 低置信度时，明确提示解析可能不完整。

### 12.2 Prompt 约束

- 根据 subject、gradeLevel 和 grade 控制讲解深度。
- 面向初高中学生和家长，语言清晰，避免过度专业术语。
- 不编造题干中没有的信息。
- 如果图片或 OCR 信息不足，先说明缺失信息，再给可推断部分。
- 输出 Markdown，方便 App 渲染。
- MVP 只针对数学和物理优化，必须保留公式、几何关系、物理过程和关键推导。

### 12.3 接口返回补充

`POST /ai/solution` 的任务结果中，explanationMarkdown 必须使用固定二级标题，顺序为：答案、解题思路、详细步骤、易错点、同类题方法、不确定提示。

每个标题下必须有正文。如果某个可选部分没有内容，例如同类题方法，正文写“暂无补充”。如果模型认为题目信息不足，“不确定提示”必须说明缺失的是题干、图片细节、公式、选项还是图形信息。

如果模型认为题目信息不足，必须在 warnings 中返回原因，并在 explanationMarkdown 的“不确定提示”中展示给用户。

## 13. MVP 验收标准

MVP 完成时至少要通过以下验收：

- 用户可以拍照或从相册导入试卷图片。
- 用户可以选择单题模式或多题模式。
- 用户可以选择错题、调整题框，并将错题保存到本地题库。
- MVP 体验和验收聚焦初高中数学、物理错题，不为小学生、大学生或非理科科目优化。
- 每道错题保存原始题图、系统增强/清理图、用户最终修复图的版本信息。
- 云端 OCR、图片增强、自动去手写失败时，不影响用户继续橡皮擦修复和打印。
- 错题详情支持橡皮擦清除残留作答痕迹；MVP 不实现恢复笔刷、自由画笔、文字标注、线图覆盖、裁剪、旋转等图片编辑功能。
- 错题库支持按数学/物理、初中/高中、年级、时间、标签筛选。
- 错题详情可以生成并查看 AI 答案解析。
- 用户可以选择多道错题生成 PDF，PDF 使用最终修复题图，并只为解答题增加作答区。
- Android 和 iOS 至少各完成一次拍照、入库、橡皮擦修复、PDF 导出的端到端验证。
